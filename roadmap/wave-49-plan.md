## Overview

The repo has 51 CLAUDE.md files auto-generated by the IDE (haiku/sonnet/opus pick). The current generation prompt produces comprehensive file-role tables, subdirectory indexes, dependency lists, and architectural flow diagrams. Investigation of `db51e123-*.jsonl` shows nested CLAUDE.md injections account for ~35k tokens (~10% of transcript), ~half of which was re-injection when the agent bounced between subtrees.

Anthropic's own guidance: **target under 200 lines per CLAUDE.md file, as longer files consume more context and reduce adherence.** Four of the seven files injected in that transcript exceed that limit — by 30–75%:

| File | Current lines | Status |
|---|---|---|
| `src/main/agentChat/CLAUDE.md` | ~350 | 75% over |
| `src/main/orchestration/providers/CLAUDE.md` | ~300 | 50% over |
| `src/renderer/components/AgentChat/CLAUDE.md` | ~280 | 40% over |
| `src/main/CLAUDE.md` | ~260 | 30% over |

The correct structural response is: **CLAUDE.md is for tribal knowledge, the graph serves derivable content on demand.** File-role tables, dependency lists, and architectural flows are all queryable via `get_architecture` / `search_graph` / `trace_call_path`. Keeping them in CLAUDE.md means paying tokens for stale hand-maintained duplicates.

Wave 49 ships two mechanisms:

1. **New generation prompt** that excludes derivable content and produces lean, gotcha-focused CLAUDE.mds. Regenerate the four over-budget files under this prompt.
2. **Gotcha-update rule/hook** so CLAUDE.mds grow organically from real encounters instead of staying frozen at generation time.

## Implementation review summary

### Confirmed state

- `claudeMdGenerator.ts` spawns `claude` CLI headless to produce each CLAUDE.md. Model is configurable (haiku/sonnet/opus).
- `claudeMdGeneratorSupport.ts` owns directory discovery, file listing, prompt building, git-diff detection for stale files.
- No current prompt constraint on length — generator happily produces 350-line files.
- No current constraint on excluding derivable content — generator includes full file-role tables because that's what well-formed tech docs look like in its training data.
- No organic update path — CLAUDE.mds are generated once then drift until explicitly regenerated.
- `claudeMdGeneration` config namespace does not exist yet.

### Gaps this wave closes

- **Generation prompt is too permissive.** Produces exhaustive documentation instead of lean tribal knowledge.
- **No organic growth.** A gotcha discovered during normal work doesn't get captured unless a human remembers to add it.
- **No lint/check enforcing size cap.** Nothing prevents CLAUDE.mds from creeping back past 200 lines over time.
- **No migration path for existing over-budget files.** Someone has to either regenerate-all (expensive, loses curated content) or groom by hand (tedious, error-prone).

## Scope

### In-scope

- New generation prompt template that excludes file-role tables, subdirectory indexes, dependency lists, call-flow diagrams. Explicit `OMIT rather than speculate` constraint.
- Regeneration of the 4 over-budget CLAUDE.mds identified above, under the new prompt.
- Rule in project CLAUDE.md instructing agents to append discovered gotchas to the nearest subsystem CLAUDE.md's `## Gotchas` section.

- Changing global (`~/.claude/CLAUDE.md`) memory files — those belong to the user.
- Removing CLAUDE.mds entirely (the proposal is lean, not gone).
- Rule-to-hook migration for existing rules (Wave 50).
- Gotcha-extraction from git history backfill (possible Wave 54+).

## Architecture

```text
generation (wave 49 onwards)
 ├─ generateForPath(dir)
 │    ├─ collectCodeSamples            (unchanged)
 │    ├─ collectInlineWarnings         ← NEW: scrape // NOTE / // WARNING / // DO NOT
 │    └─ buildLeanPrompt               ← NEW: binding constraints
 │         ├─ EXCLUDE derivable content
 │         ├─ INCLUDE only non-derivable
 │         └─ OMIT rather than speculate
 └─ writeClaudeMd(dir, result)
       └─ validateSize(<= 200 lines)   ← NEW: reject and retry with stricter prompt

organic growth (wave 49 onwards)
 └─ Stop hook on session end
      ├─ detectBugFixInDiff()
      └─ if fixed → prompt agent to consider CLAUDE.md gotcha update

lint
 └─ npm run lint:claude-md             ← NEW: pre-commit check for size cap
```

**Key design calls:**

- Prompt must contain the phrase **"OMIT rather than speculate"** — without it, the generator fills empty space with invented gotchas.
- Lean prompt should explicitly tell the generator to quote inline comments as supporting evidence. This anchors gotchas in code, not imagination.
- The Stop-hook prompt is a *nudge*, not an enforcement. Asking the agent to consider is different from forcing an update. Over-hooking breeds friction.
- Size-cap lint should fail on 200+ lines but allow grandfathered files (via YAML frontmatter exemption) until they're regenerated. Don't block work on day-one.

### Subagent briefing

- **Read first:** `claudeMdGenerator.ts`, `claudeMdGeneratorSupport.ts`, two or three current CLAUDE.mds in the repo to see what "too much" looks like today.
- The prompt template MUST include:
  - Explicit EXCLUDE list: file-role tables, subdirectory indexes, import/export dependency lists, architecture flow diagrams.
  - Explicit INCLUDE list: gotchas (load-bearing patterns, design decisions with rationale, "don't refactor this" warnings).
  - The phrase "OMIT rather than speculate" — this is the single most important guardrail.
  - Target size: under 150 lines (leaves headroom under the 200-line cap).
  - Directive to quote inline warnings when relevant as supporting evidence.
- Do NOT have the prompt say "be concise" without the EXCLUDE list — models will just reformat the derivable content into tighter tables.
- Test with haiku first — if haiku can produce decent lean output under this prompt, sonnet and opus will too.

### Subagent briefing

- **Read first:** `hookInstallerCommands.ts:12-33`, Wave 48's `graphUsageLogger.ts` pattern.
- The nudge is passive: emit a structured message, don't block session completion.
- Detection heuristic: session changed existing files (not only adds), and the commit message contains `fix`, `bug`, `issue`, `gotcha`, `regression`, or similar. Over-trigger is better than under — the agent can dismiss.
- Size-cap lint allows grandfathering via a comment marker so we don't break CI on files that haven't been regenerated yet. The expectation is Wave 49 + manual sweep brings everything under cap over time.
- Rule in CLAUDE.md is prescriptive. Be explicit: "MUST append before completing the task."

## Subagent execution model

- **Model:** `sonnet`
- **Isolation:** sequential on `master`
- **Test policy:** scoped vitest per phase; parent runs full suite + `lint:claude-md` at wave close
- **Lint policy:** size-cap lint grandfathers non-regenerated files only if they carry the marker
- **Commit policy:** conventional commits, one per phase (Phase B is the exception: one commit per regenerated file)
- **Scope discipline:** do NOT regenerate files outside the 4 Phase-B targets. Do NOT modify global `~/.claude/CLAUDE.md`.

### Phase dispatch order

1. **Phase A** — lean generation prompt (foundation)
2. **Phase B** — regenerate 4 files (depends on A)
3. **Phase C** — gotcha hook + size cap (independent of A/B, can run in parallel with B)
4. **Phase D** — integration + docs

## Risks

| Risk | Mitigation |
|---|---|
| Lean prompt produces too-skeletal output that loses real value. | Human review required on Phase B. If quality is poor, iterate the prompt in Phase A before proceeding. |
| Generator invents gotchas despite "OMIT rather than speculate". | Fixture tests include adversarial inputs (dirs with no inline warnings) — expect empty Gotchas section in that case. |
| Stop-hook over-nudges and becomes annoying. | Trigger heuristic deliberately conservative (commit message keywords + file modifications). Telemetry measures nudge-vs-follow-through rate for tuning. |
| Size-cap lint breaks existing CI. | Grandfather marker allows existing files to pass until regenerated. Add the marker to all 40+ not-yet-regenerated files as part of Phase D. |
| Regenerated CLAUDE.mds lose architectural context that was manually curated. | Phase B requires diff review and merge — not blind overwrite. |
| Agents ignore the gotcha-maintenance rule. | Telemetry will reveal follow-through rate. Wave 50 can convert to enforcement if soft rule isn't sufficient. |
